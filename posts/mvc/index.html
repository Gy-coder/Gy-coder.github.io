什么是MVC？

  • 每个模块都可以写成三个对象 M + V + C

    1. M为Model，数据模型，用于操作所有的数据
    2. V为View，视图模型，负责所有的UI界面
    3. C为Control，控制器，负责其他的内容

    下面是MVC的代码示例

      const m  = {
      data : {
          n = parseInt(localStorage.getItem('x')) || 100
      },  
      create(){
          //增 
      },
      delete(){
          //删    
      },
      update(){
          //改            
      },
      get(){
          //查 
      }   
  }   

      const v = {
      el:null,

  }

eventBus

  • eventBus主要有on trigger off方法

      class EventBus{
      constructor() {
          //
          this._eventBus = $(window)          
      }
      on(EventName,fn){
          //绑定事件
          return this._eventBus.on(EventName,fn)
      }
      off(EventName,fn){
          return this._eventBus.off(EventName,fn)
      }
      trigger(EventName,data){
          return this._eventBus.trigger(EventName, data)

      }   
  }

表驱动编程

  • 表驱动编程是指当看到大批类似但不重复的代码，然后看看那些数据是重要的数据，把这些重要的数据做成哈希表，代码就会简化

  • 例如

      const c = {
      init(){
        //c的初始化方法    
      },
      event:{
        //事件字符串
        'click .add':'add',
        'click .subtract':'subtract',
        'click .divide':'divide',
      },
      add(){
          //事件函数
      },
      subtract(){},
      divide(){},
      autoBindEvent(){
          //事件绑定函数
      }
  }

    就是一个表驱动编程的例子，把事件写在一个event哈希表中，简化了多次绑定事件的代码

如何理解模块化

  • 模块化可以使一个项目的不同功能之间的代码彼此分开，互不影响，各司其职，从而能够解决维护性难，代码黏着度高等现象，易于维护和修改

  • 通过MVC的模块化思路也可以将同一个项目的不同角色分开 用M来表示数据 用V来表示视图 用C来表示控制器 在具体项目实现的时候可以使得代码的逻辑更加的清晰
