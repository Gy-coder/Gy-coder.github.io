Vue中computed和watch的区别

May 13, 2020

什么是Computed？

Computed就是计算属性

用法:

const Vue = window.Vue
let obj = {n:0}
new Vue({
  data(){
    return {
      n:obj.n
    }
  },
  computed:{
    stringZero(){
      let hash = {0:'零'}
      return hash[obj.n]
    }
  },
  template:`<div>{{n}}
      <div>{{stringZero}}</div
</div>`
}).$mount('#app')

什么是Watch？

Watch就是侦听器，其作用就是侦听属性的变化

语法

  1. 语法1

    const Vue = window.Vue
    
new Vue({
  data(){return {n:0}},
  template:`
    <div>
        {{n}}
        <button @click="add">+1</button>        
    </div>`,
  methods:{
    add(){
      this.n+=1
    }
  },
  watch:{
    'n'(){
      console.log('n變了')
    }
  }
}).$mount('#app')

  2. 語法2

    const Vue = window.Vue
    
    
new Vue({
  data(){return {n:0}},
  template:`
    <div>
        {{n}}
        <button @click="add">+1</button>        
    </div>`,
  methods:{
    add(){
      this.n+=1
    },
   appear(){
      console.log('n 變了')
   }
  },
  watch:{
    'n': 'appear'
  }
}).$mount('#app')

  3. 语法3

    const Vue = window.Vue
new Vue({
  data(){
    return {n:0}
  },
  template:`<div>
    {{n}}
    <button @click="add">+1</button>
  </div>`,
  methods:{
    add(){this.n += 1}
  },
  watch:{
    'n':[function(){
      console.log('n变了')
    },
        function(){
          console.log('great')
        }]
  }
}).$mount('#app')

  4. 语法4

    new Vue({
  template:`<div>
      {{n}}
    <button @click="add">+1</button>
    </div>`,
  data:{'n':0},
  methods:{
    add(){
      this.n += 1;
    },
  },
  watch:{
    'n':{
      handler(){
        console.log(1)
      }
    }
  }
}).$mount('#app')

  5. immediate

    immediate属性的功能是在内容被创建时就监听一次(值为true时)，属性的值为true或false

    new Vue({
  template:`<div>
      {{n}}
    <button @click="add">+1</button>
    </div>`,
  data:{'n':0},
  methods:{
    add(){
      this.n += 1;
    },
    show(){
      console.log(2)
    }
  },
  watch:{
    'n':{
      handler:"show",
      immediate:true
    }
  }
}).$mount('#app')

    实例

    在实例中 n刚被创建出来 2即刻打印

  6. deep属性

    deep属性的值为true或false，该回调会在任何被侦听的对象的property改变时被调用，不论其被嵌套多深

    实例1

    在实例1中没有加deep属性，默认deep属性是false，所以+1操作虽然可以实现，但watch内的handler却不会出发，因为watch并没有认为a对象改变(a在内存中的地址没有变，所以对象没有变)

    实例2

    在实例2中，deep属性为true，当a中的b改变时，会认为a改变了，从而打印出1

watch的异步执行

实例

在这个实例中 撤销操作会出现bug，因为当n改变时，会调用watch.n，由于watch.n是异步调用，所以 this.inUndoMode = false会先与watch.n执行，因此会出现bug

bug的解决

使用this.$nextTick之后，会让watch.n异步代码先执行，执行完毕后再执行this.$nextTick中的代码

computed和watch的区别

  1. computed就是计算属性的意思，watch是侦听的意思

  2. computed是用来计算出一个值的，使用时不需要加括号，并且会根据依赖缓存

  3. watch会在某个属性变化时，执行一个函数
