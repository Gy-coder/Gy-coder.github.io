作用域和闭包

January 10, 2020

作用域

{
    let a = 1
}

如上述代码所示，一个{}就代表了一个作用域，a变量只能在作用域内访问到

每一个函数都能够创建一个作用域

例如:

function f1(){
    let a = 1
}
console.log(a)
//Uncaught ReferenceError: a is not defined

Q:上述代码中a不存在的原因是否是f1没有执行？

A:即使函数f1执行，a一样不存在

function f1(){
    let a = 1
}
f1()
console.log(a)

Q:打印出多少？

A:a还是不存在，因为无法访问到作用域内的a

全局变量和局部变量

  1. 全局变量

    • 在顶级作用域中声明的变量成为全局变量
    • 在window属性内的都是全局变量

  2. 局部变量

    • 其他都是局部变量

作用域规则

  • 如果有多个作用域中存在同一个变量a

    1. 查找a的声明时，就向上取最近的作用域
    2. 查找a的过程与函数的执行无关
    3. 但a的值与函数的执行有关

  • 例如

    function f1(){      //1.定义f1
  let a = 1       //3. 定义a变量赋值为1
  function f2(){    //4. 定义f2
    let a = 2      //8.在新的作用域内定义a并赋值为2
    function f3(){     //9.定义f3
        console.log(a)   //12.打印a，根据作用域规则，找向上最近的作用域，为f2作用域，值为22
    }
    a = 22     //10.重新赋值f2作用域内的a，值为22
    f3()       //11.f3执行
}
  console.log(a)   //5. 打印a 值为1
  a = 100          //6.重新赋值a为100
  f2()             //7.f2执行
}
f1()    //f1执行

闭包

  • 如果说一个函数用到了外部变量，那么这个函数加这个变量，就构成了闭包

  • 闭包的实例

    let a = 2
function f3(){
console.log(a)
}

    在这段代码中，函数f3用到了外部变量a，因此就够成了闭包
